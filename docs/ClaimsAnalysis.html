ClaimsAnalysis¶

The ClaimsAnalysis module contains the ClaimsAnalysis class, which is a parent class with attributes that are inherited by its child classes such as Redamits and Cost. As it’s namesake implies, all child classes of ClaimsAnalysis pertain to claims. ClaimsAnalysis objects can be viewed as sub-objects of Paletable objects. As such they do not return DataFrames on their own; they simply produce queries that can be joined with Paletable objects to append columns onto DataFrames or change the context of Paletable objects entirely.

class ClaimsAnalysis.ClaimsAnalysis¶

Bases: object

Being a parent class, like Paletable, ClaimsAnalysis is not directly interacted with. Instead, its attributes and methods are inherited by its child classes. More information can found on these attributes and methods below:

callback¶

Used by ClaimsAnalysis objects to initialize methods that create additional columns for the DataFrame. See Readmits.Readmits.calculate() and Cost.Cost.caclulate()

alias¶

The alias that will be used for the ClaimsAnalysis objects SQL query.

date_dimension¶

The DateDimension object relevant to the ClaimsAnalysis object.

filter¶

The attribute responsible for indicating whether a ClaimsAnalysis object needs to be constrained by state.

join_sql¶

The methods used produce the SQL query for a ClaimsAnalysis object.

Methods:

sql()¶

The SQL query that the ClaimsAnalysis child class uses to pull dataframes.

This can be called allowing an analyst to view the SQL query in question.

Parameters

self – None - no input required.

Returns

Returns a text string containing the SQL query run by the ClaimsAnalysis child class.

Return type

str

Example

Create a Cost object:

>>> cost = Cost.inpatient()

Return the query as text:

>>> print(cost.calculate().sql())

Create a Readmit object:

>>> readmits = Readmits.allcause(30)

Return the query as text:

>>> print(readmits.sql())
prepare()¶

The prepare function is a simple “pass” function. Meaning that when this function is executed, it passed to the prepare function specific to the child class.

Note

See prepare() or prepare() to view the specific versions of this method.

calculate()¶

The calculate function is a simple “pass” function. Meaning that when this function is executed, it passed to the prepare function specific to the child class.

Note

See calculate() or calculate() to view the specific versions of this method.

apply_filters()¶

The apply filters function is not directly interacted with by the user, but is called automatically when a query produced by a ClaimsAnalysis object will be joined to a Paletable object’s query that is constrained by state. This ensures that query produced by the ClaimsAnalysis object is also constrained by state.

join_inner() → str¶

The join_inner method is responsible for producing a join conidition that is appended to a Paletable object’s query ensuring that the query produced by a ClaimsAnalysis object is properly integrated. As this method’s namesake implies this function produces an inner join condition.

join_outer() → str¶

The join_inner method is responsible for producing a join conidition that is appended to a Paletable object’s query ensuring that the query produced by a ClaimsAnalysis object is properly integrated. As this method’s namesake implies this function produces an outer join condition.
